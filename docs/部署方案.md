# TNFD 平台部署指南

## 方案一：Vercel + Railway（MVP 推荐）

### 前端部署（Vercel）
```bash
cd ~/Desktop/TNFD/frontend
npm install
npm run build
vercel deploy --prod
```

### 后端部署（Railway）
```bash
cd ~/Desktop/TNFD/backend
# 1. 安装 Railway CLI
npm install -g @railway/cli

# 2. 登录并创建项目
railway login
railway init

# 3. 部署
railway up
```

### 环境变量配置
```bash
# Railway Dashboard 设置
QWEN_API_KEY=sk-xxx
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

## 方案二：阿里云 ECS + Docker（生产环境）

### 1. 购买 ECS 实例
- 配置：2 核 4G（初期够用）
- 系统：Ubuntu 22.04
- 地域：北京/上海（根据用户位置）

### 2. 安装 Docker
```bash
# SSH 登录服务器
ssh root@your-ecs-ip

# 安装 Docker
curl -fsSL https://get.docker.com | bash
systemctl enable docker
systemctl start docker

# 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

### 3. 创建 Docker Compose 配置
```yaml
# docker-compose.yml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - QWEN_API_KEY=${QWEN_API_KEY}
    volumes:
      - ./data:/app/data
    restart: unless-stopped
  
  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
    depends_on:
      - backend
    restart: unless-stopped
```

### 4. 构建并启动
```bash
docker-compose up -d
docker-compose logs -f
```

### 5. 配置 Nginx（可选）
```nginx
# /etc/nginx/sites-available/tnfd
server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://localhost:3000;
    }
    
    location /api {
        proxy_pass http://localhost:8000;
    }
}
```

## 方案三：本地部署（数据敏感客户）

### 一键部署脚本
```bash
# deploy-local.sh
#!/bin/bash
set -e

echo "=== TNFD 本地部署脚本 ==="

# 检查 Python
python3 --version

# 安装后端依赖
cd backend
pip3 install -r requirements.txt

# 启动后端
nohup uvicorn main:app --host 0.0.0.0 --port 8000 &

# 安装前端依赖
cd ../frontend
npm install
npm run build

# 启动前端（生产环境用 PM2）
pm2 start npm --name "tnfd-frontend" -- start

echo "=== 部署完成 ==="
echo "后端：http://localhost:8000"
echo "前端：http://localhost:3000"
```

## 数据隐私合规

1. **数据不出境**：所有数据存储在阿里云国内节点
2. **敏感数据本地化**：供应商坐标数据可选择本地存储
3. **LLM 调用**：使用阿里云百炼（DashScope），数据不出境
4. **日志审计**：记录所有数据访问日志，满足监管要求
